MongoDB Query Reference

All commands and operators supported in the Atlas Shell terminal.

This is a simulated shell. It runs against synthetic in-memory documents — no real database, no writes, no network. Behavior matches MongoDB Atlas Shell for the supported subset below. Commands not listed here are not implemented.
mongo-compatible — works exactly like the real MongoDB shell limited — supported but with caveats vs real MongoDB

Database Commands

CommandDescription

Query Methods

MethodDescription
db.<col>.find(filter, projection) Return all documents matching filter. Optionally shape output with projection. mongo-compatible db.calls.find({"orgId": "abc123"}) db.calls.find({}, {"orgId": 1, "ts": 1, "_id": 0})
db.<col>.findOne(filter, projection) Return the first document matching filter, or null. mongo-compatible db.users.findOne({"email": "alice@co.com"})
db.<col>.countDocuments(filter) Return the count of documents matching filter. mongo-compatible db.errors.countDocuments({"level": "error"})
db.<col>.distinct("field") Return an array of distinct values for field across all documents. mongo-compatible db.calls.distinct("organizationId")

Cursor Modifiers

Chain onto find() in any order.

ModifierDescription
.sort({field: 1}) 1 = ascending, -1 = descending. Multiple fields supported. mongo-compatible db.logs.find({}).sort({"ts": -1})
.limit(N) Return at most N documents. mongo-compatible db.logs.find({}).sort({"ts": -1}).limit(10)

Filter Operators

Comparison
$eqEqual to mongo-compatible {"status": {"$eq": "active"}}
$neNot equal to mongo-compatible {"status": {"$ne": "closed"}}
$gtGreater than mongo-compatible {"retries": {"$gt": 3}}
$gteGreater than or equal mongo-compatible {"score": {"$gte": 100}}
$ltLess than mongo-compatible {"latencyMs": {"$lt": 500}}
$lteLess than or equal mongo-compatible
$inMatches any value in array mongo-compatible {"level": {"$in": ["warn","error"]}}
$ninMatches none of the values in array mongo-compatible
Logical
$and All conditions must match mongo-compatible {"$and": [{"level":"error"},{"retries":{"$gt":2}}]}
$or At least one condition matches mongo-compatible {"$or": [{"level":"error"},{"level":"fatal"}]}
$nor None of the conditions match mongo-compatible
$not Negates a field-level operator expression limited {"status": {"$not": {"$eq": "active"}}}
Element / Eval
$exists Field exists (true) or is absent (false). mongo-compatible {"deletedAt": {"$exists": false}}
$regex String matches a regular expression. Use with $options for flags. mongo-compatible {"email": {"$regex": "@murmuration\\.org$"}} {"msg": {"$regex": "timeout", "$options": "i"}}

Projections

Second argument to find() and findOne(). Cannot mix inclusion and exclusion (except _id).

ModeSyntaxBehavior
Inclusion {"field": 1} Return only the specified fields. _id is included by default unless you add "_id": 0. mongo-compatible db.users.find({}, {"email": 1, "orgId": 1, "_id": 0})
Exclusion {"field": 0} Return all fields except the specified ones. mongo-compatible db.users.find({}, {"password": 0, "token": 0})
Dot notation {"address.city": 1} Project a nested field. mongo-compatible db.orgs.find({}, {"billing.plan": 1})

Extended JSON Types

Use MongoDB extended JSON syntax in filters to match ObjectId and date fields.

TypeSyntaxExample filter
ObjectId {"$oid": "..."} {"_id": {"$oid": "65f3a1b2c4d5e6f7a8b9c0d1"}} mongo-compatible
ISODate {"$date": "..."} {"createdAt": {"$gt": {"$date": "2024-01-01T00:00:00Z"}}} mongo-compatible

Dot Notation

Access nested fields in filters and projections using dot notation. Works to any depth.

"address.city" Matches doc.address.city mongo-compatible db.users.find({"address.city": "New York"})
"billing.plan.tier" Matches doc.billing.plan.tier mongo-compatible

Not Supported

These real MongoDB features are not implemented in this shell.

aggregate()No aggregation pipeline. Use countDocuments() and distinct() for grouped summaries.
.skip(N)No cursor skip. Use .limit() with .sort() instead.
$elemMatchNo array element matching.
updateOne(), insertOne(), deleteOne()No write operations — this is a read-only investigation environment.
createIndex()No index management.
Write operationsAll data is synthetic and read-only. Queries never modify documents.
